# Anatomy of a Robot

[Source](https://motion.cs.illinois.edu/RoboticSystems/AnatomyOfARobot.html)
-----------------------

- Robots can be thought of as a "computer on wheels"
- From a hardware perspective, rigid sections (links) are connected by moveable "joints" at articulation points
- The "brain" of a robot consists of software running on the computer which processes inputs from sensors, and computes
output which drive actuators. This communication is done via physical wires, or wireless communication

# Links & Joints

-----------------------

- Links offer structural support and the protection of internal parts
- Joints allow for rotation or translation

# Sensors

-----------------------

- Sensors measure physical quantity in the real world and encode these signals into electric (digial) form

| Type | Example | Purpose |
| :--- | :---: | ---: |
| Proprioceptive | Joint encoders, Joint limit cotact switches | Measure link position relative to other links |
| Interial | gyro, accelorometer, IMU | Measure velocity, acceleration, or position of a link |
| Tactile | Contact switch, force/torque sensor (load cell), scale | Detect external forces on robot body
| Visual | Camera, depth sensor | Measure light intensity from surroundings
| Other | GPS, Sonar | N/A
- Sensors are typically attached to the links and move with the robot. The raw data produced by sensors
usually needs to be processed by proception algorithms
- Position and orientation of the sensor on a link is very important

# Actuators and Transmissions

-----------------------

- An actuator generates force/torque signal when provided with electrical signal
    - Common types: Electric motors, hydraulic actuators, etc
- A transmission applies the force / torque of an actuator to robot links 
    - Common types: Gearing, cables
- Actuators either revolve around an axis or translate around an axis
- If an actuator is pushed beyond its safety limits, such as burning out or blowing seals

# Cognitive Architectures

-----------------------

- Not "conciousness" like humans or animals, but a way to process surroundings and make decisions
- Resides in the "mind" of a robot, body & external world form the information processing environment

#### Sense-plan-act Framework
- Role of robot mind is to receive sensor information (sense), process (plan), and then output commands to the actuator (act)

#### Perception, Planning and Control; hierarchial architectures
- In the above framework, "plan" component can be broken down into 3 components
    - Perception: From sensor data and memory from the past, constructure a representation of all the robot's knowledge, and then make a decision. If no data, make a "best guess"
    - Planning: Use robot knowledge and dynamic models to decide
    - Control: At a high feedback rate, execute reference motions which are generated by outputting actuator signals
